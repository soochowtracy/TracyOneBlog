Think in Block

2015-06-07
iOS

Block Brief

Block是什么

苹果官网说法：
A block is an anonymous inline collection of code, and sometimes also called a “closure”.

闭包就是能够读取其它函数内部变量的函数 —阮一峰

普通函数能使用哪些变量：

  • 局部变量
  • 参数
  • 静态变量
  • 静态全局
  • 全局变量

多次调用之间能够传递值得变量：

  • 静态变量
  • 静态全局
  • 全局变量

BLock的用法和功能

语法

void (^)(void){
};
与C语言函数相比没有函数名，带有^；可省略很多部分。

变量

void (^var)(void)；
与C语言函数指针相比只是把*改成了^,Block变量和普通变量作用完全相同。

  1. 通过typedef可以简化
  2. 调用和C语言一样，变量名+小括号
capture
  1. Block可以使用在它之前声明的局部变量，因此在执行Block时，即使已经改变了值，也不影响Block中截获的值。
  2. 一般情况，Block只能保存声明时的瞬间值，保存后就不能修改了；如果需要改变则要附加__block.
  3. 如果截获的是oc对象，使用是没有问题的，但是如果要重新赋值，则也必须加__block。

Block,__block存储域

Q:Block超出变量作用域可存在？

A:

  1. 配置在全局变量中得Block，在变量作用域之外也可以通过指针安全的使用
  2. 在栈上得Block则是通过复制到堆上来解决的
  3. 而block变量用 forward可以实现无论配置在哪都可以正确地访问 __block变量
  4. 在ARC下，大多数情况下编译器会恰当的判断，自动将Block从栈上复制到堆上。（autorelease）

Global:

  1. 在全局变量声明区
  2. Block声明中没有截获局部变量时
  • 自动copy：

Block作为函数的返回值时

  • 手动copy：

向方法或函数的参数中传递Block（但是可以在函数中适当地copy就不必在传递前copy）（GCD中得）

调用Copy时不同的Block对象究竟发生了什么：

  1. _NSConcreteStackBlock ：从栈复制到堆
  2. _NSConcreteGlobalBlock ：什么也不做
  3. _NSConcreteMallocBlock ：引用计数增加

调用Copy时不同的__block对象究竟发生了什么：

  1. 栈 ：从栈复制到堆，并且被Block持有
  2. 堆 ：被Block持有

Q:__block变量的结构体成员forwarding？

A:为了能够在不同位置的__block变量或者栈变量都能访问的是同一个变量

capture

正常情况下，C语言是不支持__strong修饰符的变量，不能很好地管理内存。但是runtime给了我们机会，

让我们知道Block什么时候复制到堆上，什么时候会废弃。so，我们就用到了copy和dispose函数。

只有在Block对象调用过copy后，Block才能持有截获的__strong类型的变量。

很容易引起循环引用，如何解决：

  1. __weak修饰符
  2. __block修饰符（nil）
  3. unsafeunretain

非ARC下 __block不会持有对象。
